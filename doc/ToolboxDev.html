ToolboxDev (class)

Imrove develop process of your toolbox
Table of Contents
How to add Properties Methods Additional information Documentation App icon Use ToolboxDev without adding Examples

How to add

To add Dev class to current project folder execute
ToolboxExtender.add('dev')

Properties

ext - ToolboxExtender object
vp - toolbox project version (from project file), char

Methods

dev = ToolboxDev([extender])
Create dev object of ToolboxDev class.
If you call ToolboxDev class with optional ToolboxExtender object extender, dev will use this object, otherwise new ToolboxExtender object will be created inside dev (ext property).
If you call ToolboxDev class with optional path to folder (extender argument), new ToolboxExtender object will be created inside dev (ext property) for extender path.
vp = gvp()
Get project version
build([version])
Build binary installation file for destribution (.mltbx or .mlappinstall) for current version or for version (optional) specified.
test([version])
Build and install current or specified version. Version setting can be useful when you test the Updater class.
setver(version)
Set specified project version.
gendoc([format], [docdir], ['TargetDir', targetdir], ['AddCredentials', addcred])
Generate html, pdf or md (beta) documentation from .mlx-files stored in docdir folder (default: './doc'). Use this method if you want to test documentation without toolbox building. You can specify format as 'html', 'pdf' or 'md' (default: pdf). If addcred is true (default: false), Toolbox Extender credentials will be added to generated .md-file. Also optionally you can specify targetdir directory where generated files will be stored.
release([version])
Release current or specified version (optional) of toolbox. This method automatically perform:
  1. Generate .html files from .mlx in doc folder
  2. Fix app icon (if app is included in project)
  3. Build installation file (see build())
  4. Commit and push project to remote (GitHub)
  5. Tag the new one commit on GitHub by current version
  6. Copy installation file path to clipboard
  7. Open in web browser GitHub release page to help your create a new release of your toolbox (don't forget to attach binary installation file from your clipboard!)
check()
Check that latest release is valid and suitable for the ToolboxUpdater routine
webrel()
Open release page on GitHub
webinstaller()
Get MATLAB command to install toolbox directly from web (read about installers)
untag(version)
Delete specified by version tag from GitHub. Use it in to delete fault release from remote

Additional information

Documentation

Store documentation as .mlx-files in doc folder. When build() or release() its will be automatically converted to .html

App icon

If your toolbox suppose any App Designer app you may wish to add icon to it (against MATLAB bug). Just add icons for your app (icon_16.png and icon_24.png) to resources folder

Use ToolboxDev without adding

If you want to use ToolboxDev without toolbox or just for Git-functionality (for project, package, etc) you can do it without adding ToolboxDev class to current folder.
To do it execute:
dev = ToolboxDev(pwd)

If you want to use ToolboxDev only to work with Git (i.e. in current folder):
dev = ToolboxDev(pwd)
dev.release('2.0') % create and push git tag and create release on GitHub
dev.untag('2.0') % delete tag form git and GitHub
dev.ext.web % open GitHub page
