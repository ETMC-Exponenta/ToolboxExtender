ToolboxUpdater (class)

Add to your toolbox ability to update itself from remote (GitHub)
Table of Contents
How to add Properties Methods Additional information How to integrate Examples

How to add

To add Updater class to current project folder execute
ToolboxExtender.add('updater')

Properties

ext - ToolboxExtender object
vr - latest remote version from internet (GitHub), char
isupd - update is available, logical
relsum - latest release notes summary, char
rel - latest release notes, char
bin - toolbox binary information, struct

Methods

TU = ToolboxUpdater([extender])
Create TU object of ToolboxStorage class.
If you call ToolboxUpdater class with optional ToolboxExtender object extender, TU will use this object, otherwise new ToolboxExtender object will be created inside TU (ext property)
ison = isonline()
Check internet is available
[data, err] = fetch()
Get information about latest release from GitHub (optional to use)
vr = gvr()
Get latest remote (GitHub) version vr
isupd = isupdate([callback], [delay])
Check update is available.
If you call this method with callback function it will perform asynchronous request for a new version and after delay seconds (default: 1) will call callback function with isupd argument. This use helps you to design responsive App Designer apps with the update feature (see an example below)
[vc, vr] = ver()
Display and get current (vc) and remote (vr) versions. If update is available show an update hint
rel = getrel()
Get full release notes of the latest release
rel = getrelsum()
Get only summary release notes of the latest release
webrel()
Open full release notes on GitHub
update([delay], [precallback], [postcallback])
Install the latest version from the remote (GitHub) if update is available.
You can specify delay in seconds before update process will be started (default: 1). precallback function will be executed just before update, postcallback will be executed after update process. This use helps you to design responsive App Designer apps with the update feature (see an example below)
installweb([dpath])
Install toolbox from the remote (GitHub). Toolbox file will be downloaded to dpath folder if specified (default: temporary directory)

Additional information

How to integrate

Firstly you need to create a release of your toolbox on GitHub with release notes and installation binary atached. You can use ToolboxDev to automate this process.
After it add to your toolbox function (or method to your app) that automatically checks that a newer version of toolbox are available, informs a user about it and allows him to install the latest version.
Examine ToolboxExtenderApp as a good reference of integrating ToolboxUpdater to an App Designer application.

Examples

Create an object of the ToolboxUpdater
TU = ToolboxUpdater
Check that update is available
isupd = TU.isupdate
Install the latest version from remote (if available)
TU.update
Force to install the latest version from the remote
TU.installweb
Check installed (current) and the latest (remote) versions
[vc, vr] = TU.ver
Also examine ToolboxExtenderApp as a good reference of integrating ToolboxUpdater to an App Designer application.
